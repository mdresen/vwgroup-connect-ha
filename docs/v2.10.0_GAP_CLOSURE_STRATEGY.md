## Competitor gaps to close in v2.10.0

20 items grouped by file impact so 3 parallel agents can each take a brand-isolated group with minimal merge conflict risk.

### Group A: VW EU field additions (10 items)

Target file: `cariad/api/vw_eu.py` (status parser) + `cariad/models.py` + `sensor.py` + `binary_sensor.py`. Surfaced during the `the VW-specific competitor library` scan; the library exposes 273 vehicle properties to our 180-ish.

- `hv_battery_min_temperature` + `hv_battery_max_temperature` — separate min/max, we collapse to one `battery_temp` today.
- `charge_max_ac_setting` vs `charge_max_ac_ampere` — distinguish user-requested setting from actual deliverable amperage.
- `auto_release_ac_connector_state` + `auto_release_ac_connector` — Born MY24+ connector auto-release.
- `optimised_battery_use` — preservation flag distinct from `battery_care`.
- `active_ventilation_state` + `active_ventilation_remaining_time` — separate from climatisation.
- `sunroof_rear_closed` + `roof_cover_closed` — rear sunroof + Cabrio top.
- `connection_state_battery_power_level` — 12V health bucket (low/normal/high).
- `last_trip_total_fuel_consumption_l` + `last_trip_total_electric_consumption_kwh` — totals, not just averages.

### Group B: SEAT/CUPRA OLA endpoints (6 items)

Target file: `cariad/api/seat_cupra.py` (status parser) + models + sensor/binary_sensor. Surfaced during the Timwun/Cupra-WeConnect-Bruno-Collection scan.

- `GET /v1/vehicles/{vin}/notifications` — in-vehicle notification list. Sensors: count + last_subject + last_severity.
- `GET /v1/vehicles/{vin}/permissions` — owner / co-driver flags. Binary sensors.
- `GET /v1/vehicles/{vin}/measurements/engines` — engine oil + coolant temperature.
- `GET /v1/vehicles/{vin}/charging/profiles` — SEAT/CUPRA flavour of the Skoda profiles we already parse.
- `GET /v1/vehicles/{vin}/charging/modes` settable — list-of-allowed-modes attribute + PUT action.
- `POST /v1/vehicles/{vin}/charging/actions/update-settings` — write charge plan as a HA service.
- `GET /v1/charging/points` (no VIN) — OLA POI catalog, extends `find_charging_stations`.

### Group C: VW NA endpoints (4 items)

Target file: `cariad/api/vw_na.py` (commands + parser) + services.yaml. Surfaced during the `zackcornelius/CarConnectivity-connector-volkswagen-na` scan.

- `GET /rrs/v1/privileges/user/{uid}/vehicle/{uuid}` — subscription + capability info.
- `POST /ss/v1/user/{uid}/challenge` + `/spin` — NA-style SPIN verification flow (different shape from EU).
- `POST /lockunlock/v1/vehicle/{uuid}` — NA-specific lock/unlock action endpoint.
- `POST /ev/v1/vehicle/{uuid}/pretripclimate/{start,stop}` + `/windowheating/{start,stop}` — NA climate naming (we use EU naming today).
