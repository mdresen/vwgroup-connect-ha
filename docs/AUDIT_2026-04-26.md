## P0 release blockers (all fixed in v1.8.0)

| # | Finding | Verified location | Fix |
|---|---|---|---|
| 1 | README claimed 10 platforms, code loaded 8 | `__init__.py:29-37` | Added `IMAGE` + `SELECT` to `PLATFORMS`; fixed `image.py` and `select.py` to use `entry.runtime_data` (was: broken `hass.data[DOMAIN]` lookup); `select.py` now uses `_attr_translation_key` |
| 2 | `iot_class=cloud_push` despite polling-only | `manifest.json` | Changed to `cloud_polling`. Returns to `cloud_push` once #57 ships real push |
| 3 | Coordinator pushed `success=True` regardless of any vehicle's actual status | `coordinator.py:182` | Per-VIN `vehicle_success: dict[str, bool]`; `is_vehicle_available(vin)` method; entity base class checks per-VIN availability |
| 4 | `unlock` warned about missing S-PIN but called the API anyway | `coordinator.py:371-374` | Raises `ServiceValidationError(spin_required)` before the call; translation key in 9 language files |
| 5 | `max_charge_current`, `seat_heating_switch`, `auto_unlock_switch` only logged or mutated internal state | `coordinator.py:435-437`, `switch.py:122-184` | Entities removed; coordinator method raises `ServiceValidationError(feature_not_supported)`. Will return when real CARIAD commands exist |
| 6 | Reverse geocoding sent vehicle GPS to Nominatim every poll | `coordinator.py:317-325` | Now opt-in via `enable_reverse_geocoding` option (default OFF); uses HA's shared aiohttp session; caches by rounded coords (~110m) |
| 7 | `quality_scale.yaml` had duplicate `comment:` keys + outdated test counts | `quality_scale.yaml` | Duplicates removed, hardcoded numbers replaced with reference to CI |

Plus 4 pre-existing mypy errors fixed: `entity_base.py:67` (bool wrap), `__init__.py:233` (bool wrap), `skoda.py:137` (None on bool field), `config_flow.py:171` (HA `domain=` kwarg type ignore).

## Risks not yet addressed

- **Pre-existing test failures:** v1.7.0 main already had Lint + Tests CI red.
  v1.8.0 fixes Lint (mypy strict) but tests remain red. Cause not yet
  visible from public CI logs. Tracked separately, does not block #60 fixes.
- **Live testing gap:** Foundation fixes are unit-tested but the per-VIN
  availability change has not been validated against real failed-poll
  scenarios. Brand Captains program (#64) addresses this systemically.
- **EU Data Act (Sep 2026):** Some current API endpoints may be deprecated
  or relocated under the standardised regime. Tracked as #59 with no work
  planned until Q3 2026.
